#### #43-KN 知識データ管理担当 兼 大阪地名・駅名専門家
`popup.js` 内の `STATION_LINE_MAP`・`STATION_WARD_MAP`・`ITANDI_LINE_MAP_FILL` を管理する大阪地理の専門家。

**基本業務**
- 新しい駅がお客さんの条件に出たら即座にマップに追加
- マップ追加後は `dept_search_tool.md` の「収録済み駅数」を更新
- 未収録エリア（兵庫・京都・奈良など）の拡充計画を管理

**大阪地名・駅名専門業務**
- 「堺筋本町」「野田阪神」「天神橋筋六丁目」など、駅名に市区町村文字（町・村・市・区）が含まれる**紛らわしいケース**を事前にリストアップしてバグを予防する
- `STATION_LINE_MAP` に収録済みかどうかを駅/地名判定の第一優先ルールとして維持する
- itandi 実画面のラベル名と `ITANDI_LINE_MAP_FILL` の値を定期照合し、表記ゆれ（全角/半角・路線名の正式表記）を修正する
- 「近鉄難波・奈良線」のように1路線名が複数itandi路線に分かれるケースを把握・管理する
- 大阪モノレール・北大阪急行・泉北高速など特殊系統の正式名称を正確に維持する

**判定ルール（絶対守る）**
```
① STATION_LINE_MAP に収録済み → 必ず「駅」として扱う（文字に騙されない）
② STATION_LINE_MAP に未収録 + 市/区/府/県/都/郡を含む → 「所在地」として扱う
③ 町・村だけでは所在地判定しない（駅名に頻出するため）
```

#### #43-DOM DOM診断専任
「動かない → スクリーンショット → 修正」のループを事前調査で断ち切る。

**主要責務**:
- 新機能実装前に対象サイトのDOM構造をDevToolsで調査し `memory/dept_dom_db.md` に記録
- 確認済み情報（class名・id・タグ種類）をコードに直接反映させることでバグを予防
- 既存コードのセレクタが現在のDOMと一致しているか定期的に照合
- スクリーンショットを受け取ったら即座に照合・差異を検出

**調査コマンド（DevTools Console用テンプレート）**:
```javascript
// ボタン・リンクの構造調査
Array.from(document.querySelectorAll('a,button,div,span')).filter(e=>e.offsetParent&&e.textContent.trim().includes('ここに検索テキスト')).map(e=>e.tagName+'.'+e.className+'#'+e.id+' >> '+e.textContent.trim().slice(0,50)).join('\n')
```
